# WMI Remote Execution Detection

## Description

Detects WMI-based remote execution which is a common lateral movement technique because it uses a legitimate Windows service, generates less obvious logs than PSExec, and is available on every Windows system by default. Detection focuses on unexpected process creation events generated by WmiPrvSE.exe on target systems.

## MITRE ATT&CK

- Tactic: Lateral Movement, Execution
- Technique: T1047 — Windows Management Instrumentation

## Platforms

Microsoft Sentinel KQL, Splunk SPL

## Microsoft Sentinel KQL Query

```kql
DeviceProcessEvents
| where Timestamp > ago(30d)
| where InitiatingProcessFileName =~ "WmiPrvSE.exe"
| where FileName !in~ (
    "WmiPrvSE.exe", "unsecapp.exe",
    "msiexec.exe", "scrcons.exe",
    "wmiprvse.exe"
  )
| where ProcessCommandLine !contains "\\REGISTRY\\"
| project Timestamp, DeviceName, AccountName,
    FileName, ProcessCommandLine,
    InitiatingProcessCommandLine,
    InitiatingProcessParentFileName
| order by Timestamp desc
```

## High Confidence Variant — WMI Spawning Scripting Engines

```kql
DeviceProcessEvents
| where Timestamp > ago(30d)
| where InitiatingProcessFileName =~ "WmiPrvSE.exe"
| where FileName in~ (
    "powershell.exe", "pwsh.exe",
    "cmd.exe", "wscript.exe", "cscript.exe",
    "mshta.exe", "regsvr32.exe"
  )
| project Timestamp, DeviceName, AccountName,
    FileName, ProcessCommandLine
| order by Timestamp desc
```

## Splunk SPL Query

```spl
index=win_* EventCode=4688 earliest=-30d latest=now
| eval parent=lower(coalesce(Creator_Process_Name, Parent_Process_Name))
| eval image=lower(coalesce(New_Process_Name, Process_Name))
| where match(parent, "wmiprvse\.exe$")
| where NOT match(image, "(wmiprvse|unsecapp|scrcons|msiexec)\.exe$")
| eval cmdline=coalesce(Process_Command_Line, CommandLine)
| table _time host user parent image cmdline
| sort 0 -_time
```

## What WMI Normally Spawns vs What Attackers Use

**Legitimate WMI children:**
- Other WMI processes (wmiprvse.exe, unsecapp.exe)
- Specific management applications in controlled environments

**Attacker WMI children (all are high confidence):**
- powershell.exe or pwsh.exe
- cmd.exe with complex command lines
- wscript.exe or cscript.exe running scripts
- Any process running from AppData, Temp, or user-writable paths

## False Positives

WMI is used legitimately by many management tools. Baseline your environment before alerting. Common legitimate WMI process creation includes some monitoring agents and endpoint management tools. The scripting engine variant has very few false positives.
